# pi-pulse documentation

This folder contains the user and contributor documentation for **pi-pulse**, a Pi footer extension that replaces the stock `"tps"` status key with live, quantitative measures of LLM provider performance.

## Index

| Document | What you'll learn |
|----------|-------------------|
| [getting-started.md](./getting-started.md) | Install the extension, build from source, and load it into Pi. |
| [metrics.md](./metrics.md) | What TPS, TTFT, and Elapsed actually mean, how they are calculated, and how to read the footer. |
| [architecture.md](./architecture.md) | How the extension is structured, why the metrics are defined this way, and the tradeoffs we made. |
| [testing.md](./testing.md) | How the test suite works and how to add deterministic tests. |
| [contributing.md](./contributing.md) | How to make changes, use changesets, and follow the project's conventions. |
